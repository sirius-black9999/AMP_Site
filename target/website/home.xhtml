About me

Hi my name is Tycho Henzen, I'm Software engineer and Game developer. My current education is focused on Software engineeering, with a specialization in Game development

I slightly prefer game development over Software engineering because i feel it's a more versatile field which allows for a much wider range of challenges and experiences

game development

I have experience using game engines like Unity3D, XNA, and LWJGL. Both in order to create games from scratch, as well as expanding upon existing projects

I also have experience using various graphics libraries, such as DirectX and OpenGL, to create some game-like projects

Finally i have created several games that use none of the techniques described above, and are coded using a "minimal libraries" setup, using mainly java's most basic components

Sadly, im not nearly as good as i would like to be in creating graphics and audio, although i can create some decent icon-sized pixel art, which has served me quite well on my journey into game development so far

Software engineering

I have experience using netbeans, eclipse, intelliJ and microsoft visual studio as IDE's and have knowledge of many of the frameworks available for java, java enterprise, and C#

i also have experience with a large selection of development tools and techniques, such as AGILE development, version control (SVN and git), jenkins, glassfish, jboss,

Teamwork

since team work is one of the most important qualities for a software engineer/game developer to have, Therefore i feel it warrants its own section. i feel it's only fair to say i'm not the best at communicating when i first join a new development team

I have a slight tendency to keep quiet in this situation, but if communication within the team is good, this shouldn't become an issue
